Implementation of the HashDB trait for a disk-backed database with a memory overlay and latent-removal semantics.

Like OverlayDB, there is a memory overlay; commit() must be called in order to write operations out to disk. Unlike OverlayDB, remove() operations do not take effect immediately. Rather some age (based on a linear but arbitrary metric) must pass before the removals actually take effect.
